02 // Ingestion

Ingest + normalize

Pull new items since last cursor. Normalize into common schema: source, sender, timestamp, content, thread_id, metadata.

Gmail search_threads(after: cursor) → sender, subject, body, labels
Slack read_channel + read_thread → author, channel, text, reactions
Granola list_meetings(since: cursor) → participants, topics, action_items
Notion query recently_modified → page_title, properties, linked_projects

Deduplicate + merge

Same thread in email and Slack? Merge into one item. Content hash skips already-processed items.

03 // Classification

Classify

Each item assigned an urgency tier. Claude evaluates sender, content, context, and timing in a single inference call.

In Normalized item + VIP list + active projects + channel filters
Out Urgency tier (P0-P3) + confidence score + reasoning

05 // Priority scoring

Priority scorer

Within each tier, items ranked by weighted composite score. Cross-source context enrichment from Granola + Notion.

  • Sender importance — VIP list, org chart, client flag
  • Deadline proximity — dates, time language, calendar
  • Topic relevance — active projects, recent meetings
  • Thread heat — reply velocity, @mentions, reactions
  • Staleness — time unanswered, follow-ups sent
06 // Action routing

Action router

Maps each classified + scored item to output actions based on tier and source.

P0 Draft response + immediate Slack alert
P1 Draft queued in Gmail, included in digest
P2 Daily digest FYI section
P3 Logged only, weekly rollup
Meetings Action items → Notion tasks w/ owner + deadline
